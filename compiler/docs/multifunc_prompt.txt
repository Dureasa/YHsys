扩展 YHC 编译器以支持多函数定义和调用。当前编译器只支持一个 main() 函数，所有变量都是 main 的局部变量。

需要实现：

1. 函数定义
   - 支持 int func(int a, int b) { ...; return expr; } 形式的函数定义
   - 函数定义在 main 之前或之后均可
   - 函数返回值类型固定为 int
   - 函数参数可变，参数作为函数内的局部变量

2. 函数调用
   - 支持 expr = func(arg1, arg2) 形式的函数调用
   - 调用可作为独立语句或嵌套在表达式中
   - 支持单级调用和嵌套调用（但不要求递归支持）

3. 函数调用约定
   - 使用标准 RV32 calling convention：a0-a5 传参，a0 返回值
   - 调用者保存：ra 和调用者需要保留的临时寄存器
   - 被调用者保存：s0 (帧指针)、ra、被修改的 s 寄存器

4. 编译流程修改
   - 前端：允许解析多个函数定义，每个函数独立创建符号表
   - IR：新增 call 指令（op="call", args: name, args[]），param 不需要新指令，调用前将参数求值到 a0-a5
   - 后端：每个函数生成独立的代码段，包含 prologue/epilogue
   - epilogue：恢复 s0/ra 后 ret，而非直接 sys_exit
   - main 的 return 仍然映射到 sys_exit

5. 编译驱动
   - 将用户函数添加到生成的汇编文件中
   - 用户函数放在 .text 段，与 main 并列
   - 确保用户函数名在汇编层面不与 OS 库函数冲突

6. 约束
   - 不要求递归（允许就行，如果自然支持了更好）
   - 不要求函数指针
   - 不要求 void 类型（所有函数返回 int）
   - 不要求前向声明（函数定义必须在调用之前出现）
   - 保持现有（只包含 main 的）代码兼容
